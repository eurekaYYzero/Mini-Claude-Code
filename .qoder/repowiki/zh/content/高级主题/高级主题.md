## 引言
本高级主题文档聚焦于 Mini-Claude-Code 的系统化演进与工程化实践，围绕"自定义工具开发""技能扩展机制""性能调优策略""系统集成方案"四大维度展开。通过对仓库中六个阶段（s01–s06）的源码进行逐层剖析，提炼出可复用的扩展点、最佳实践与风险控制手段，并给出与 CI/CD、IDE 扩展、监控系统对接的实战建议。

**更新** 新增错误恢复和弹性机制作为高级主题的重要组成部分，涵盖智能错误分类、指数退避重试策略、输出恢复等高级功能。

## 核心组件

- 微压缩（每轮）：将较旧的工具结果替换为占位摘要，保留最近若干条，避免无意义重复。

### 错误恢复与弹性机制（新增）

**更新** 新增完整的错误恢复与弹性机制，提供系统级的容错能力。

#### 智能错误分类器
- **错误分类体系**：将错误分为 retryable（可重试）、context_overflow（上下文溢出）、fatal（致命错误）、unknown（未知）四类。
- **HTTP 状态码识别**：针对 429、503、529、408、504 等可重试状态码进行分类。
- **上下文溢出检测**：识别 413 状态码和 "prompt_too_long"/"overlong_prompt" 关键字。
- **网络错误处理**：识别 ECONNRESET、ETIMEDOUT、ENOTFOUND 等网络错误关键字。
- **Retry-After 支持**：从 API 响应头提取等待时间，实现精确的等待策略。

#### 指数退避重试策略（新增）
- **p-retry 集成**：基于 p-retry 库实现智能重试，支持指数退避和 Full Jitter 策略。
- **可配置参数**：最大重试次数、最小/最大等待时间、退避因子、随机抖动开关。
- **默认配置**：最多 3 次重试，1 秒到 30 秒指数退避，启用随机抖动。
- **错误处理策略**：
  - fatal 错误：立即终止，抛出 AbortError
  - context_overflow 错误：立即终止，触发上下文压缩
  - retryable/unknown 错误：按指数退避策略重试
  - Retry-After 头部：尊重服务器建议的等待时间

#### 输出截断恢复管理（新增）
- **最大恢复次数**：限制连续输出截断的恢复尝试次数，防止无限循环。

- **状态跟踪**：维护恢复状态，包括 max_tokens 恢复次数和反应式压缩尝试状态。
- **自动重置**：正常结束时重置恢复计数器，确保下次对话从初始状态开始。

## 性能考量
- 工具调用与 I/O：文件读写与命令执行是主要开销来源，应尽量减少不必要的重复读取与长时间运行命令。
- 上下文长度控制：通过微压缩与自动压缩有效降低 token 使用，避免超出模型上下文上限。
- 并发与超时：命令执行设置超时，避免阻塞；文件系统操作使用异步 API。
- 类型与健壮性：使用 TypeScript 接口替代 any，减少运行时错误与异常分支带来的性能损耗。
- **错误恢复成本控制**：指数退避策略避免过度重试，Retry-After 头部尊重服务器负载均衡策略，防止雪崩效应。
- **输出恢复限制**：最大恢复次数限制防止无限循环，确保系统资源不会被过度消耗。

**更新** 新增错误恢复和输出恢复的性能考量，包括重试策略的成本控制和恢复次数限制。

## 故障排除指南
- 工具调用失败
  - 现象：模型调用工具但未得到对应 tool_result。
  - 排查：确认工具清单与处理器映射一致；检查工具调用块的 tool_use_id 是否正确关联。
  - 参考：多轮对话中工具调用与结果回写的严格配对要求。
- 文件路径越界
  - 现象：抛出路径逃逸错误。
  - 排查：检查输入路径是否相对且位于工作区之内；避免绝对路径与上层目录引用。
- 命令注入风险
  - 现象：命令执行存在注入隐患。
  - 排查：对命令参数进行白名单校验与转义；必要时在沙箱环境中执行。
- 上下文溢出
  - 现象：token 数过高导致截断或报错。
  - 排查：启用自动压缩；减少非必要文本回传；优先保留 read_file 输出。
- 技能加载异常
  - 现象：技能名称未知或 Frontmatter 解析失败。
  - 排查：确认技能目录结构与文件命名；检查 YAML 格式与元信息完整性。
- **API 错误处理**
  - 现象：模型调用失败，出现各种 HTTP 状态码错误。
  - 排查：使用 classifyError 函数检查错误分类；对于 retryable 错误自动重试；对于 context_overflow 错误触发压缩；对于 fatal 错误立即终止。
- **输出截断问题**
  - 现象：模型响应被 max_tokens 截断，输出不完整。
  - 排查：检查 MAX_OUTPUT_RECOVERY_ATTEMPTS 配置；确认续接消息正确注入；验证恢复状态管理。
- **重试策略失效**
  - 现象：API 调用频繁失败但未正确重试。
  - 排查：检查 withApiRetry 配置参数；验证 Retry-After 头部处理；确认错误分类准确性。

**更新** 新增错误恢复和弹性机制相关的故障排除指南，包括 API 错误处理、输出截断问题和重试策略失效的排查方法。
